\# Security Policy



\## Supported Versions



Security updates are provided for the latest production version of this project.



| Version | Supported |

|---------|-----------|

| Latest  | Yes       |

| Older versions | Limited support |



\---



\# Reporting a Security Vulnerability



If you discover a security vulnerability in this project, please report it privately.



Do not create a public GitHub issue for security vulnerabilities.



Please include:



\- Description of the vulnerability

\- Steps to reproduce the issue

\- Potential impact

\- Affected components

\- Suggested mitigation if available



\---



\# Security Response Process



After receiving a vulnerability report:



1\. The report will be reviewed.

2\. The vulnerability will be validated.

3\. The impact will be assessed.

4\. A fix will be prepared.

5\. Security updates will be released.



\---



\# Security Practices



This repository follows DevSecOps security practices:



\- Pull Request based development

\- Automated CI validation

\- Secret scanning

\- Static Application Security Testing (SAST)

\- Dependency vulnerability scanning

\- Software Bill of Materials (SBOM)

\- GitHub Actions SHA pinning

\- Continuous dependency monitoring



\---



\# Responsible Disclosure



Security researchers are encouraged to report issues responsibly.



Please allow reasonable time for investigation and remediation before public disclosure.



\---



\# Security Contact



For security-related reports, use the private security reporting mechanism provided by GitHub.
